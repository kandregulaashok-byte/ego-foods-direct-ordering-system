1. PRODUCT SUMMARY
------------------
Kitchen OS is a Windows desktop dashboard for Ego Foods, an Indian restaurant.
It is built for the owner/counter staff to manage:

- WhatsApp paid orders
- Swiggy past-order imports
- counter/offline sales
- menu and recipe mapping
- inventory and batch tracking
- expenses
- cash ledger
- daily summary
- completed sales grouped by source/item
- WhatsApp open/closed toggle
- printer status toggle

The current good UI is the local desktop layout at:
http://127.0.0.1:5173/

It is NOT the same thing as the Vercel WhatsApp backend dashboard.
The local desktop app is the primary dashboard experience.

10. WHATSAPP BACKEND BRIDGE
---------------------------
The desktop app can connect to a hosted WhatsApp backend through:
src/lib/kitchenApi.js

Expected backend endpoints:
GET /api/kitchen-os/orders
POST /api/kitchen-os/orders
GET /api/kitchen-os/settings
POST /api/kitchen-os/settings

Required request header:
x-kitchen-token: <VITE_KITCHEN_API_TOKEN>

Desktop order polling:
- App.jsx polls every 10 seconds when Kitchen API is configured
- new paid WhatsApp orders trigger alarm
- setOrders merges backend WhatsApp orders with local Swiggy/counter orders

WhatsApp ON/OFF:
- Header button calls updateKitchenSettings
- Sends is_open and closed_message to backend
- When off, bot should reply:
  "Orders are currently not being taken. If we resume in some time, we will update you."

-----------------

- electron/swiggyImporter.js
- electron/main.js
- electron/preload.cjs
- src/components/SwiggyImportPanel.jsx
- src/lib/swiggyBridge.js

Purpose:
- import Swiggy Partner past orders
- save imported orders locally
- generate daily CSV files
- generate item-count CSV files
- merge imported orders into Completed Sales/Sales views

Settings:
- mobileOrRestaurantId
- restaurantId
- password
- autoEnabled
- intervalMinutes

Default interval:
- 15 minutes

Storage:
Electron userData path:
%APPDATA%\kitchen-os\swiggy-import

Inside that folder:
- settings.json
- state.json
- orders.json
- exports\
- browser-profile\

Password storage:
- uses Electron safeStorage when available
- falls back to base64 if safeStorage unavailable
- UI should warn when storageWarning is true

Import URL:
https://partner.swiggy.com/food/pastorders/{restaurantId}

Login:
- uses persistent browser profile
- tries password login if credentials saved
- if OTP/captcha appears, visible browser allows manual completion

Daily CSV:
orders_YYYY-MM-DD.csv

Columns:
- scrapeSeenAtIso
- fullOrderId
- total
- itemsDetail

Item-count CSV:
item_counts_YYYY-MM-DD.csv

Columns:
- item
- quantity
- orders

Deduplication:
- by fullOrderId

Imported desktop order shape:
- id: swiggy-{fullOrderId}
- source: swiggy
- status: completed
- payment_confirmed: true
- pickup_code: last 4 digits of Swiggy order ID
