PRODUCT GOAL
============
Build Kitchen OS for Ego Foods: a production-grade Windows desktop application for a restaurant owner/counter staff member to manage orders, Swiggy imports, WhatsApp orders, counter sales, inventory, expenses, cash ledger, and daily summaries.

The user must be able to stand at the counter, open the desktop shortcut, and run the restaurant order workflow without technical knowledge.

INSTALL FROM SCRATCH
====================
1. Install prerequisites:
   - Windows 10/11
   - Node.js
   - Git

2. Clone:
   git clone https://github.com/kandregulaashok-byte/ego-foods-direct-ordering-system.git

3. Enter folder:
   cd ego-foods-direct-ordering-system

4. Use latest main:
   git checkout main
   git pull

5. If needed, restore good layout:
   git checkout local-desktop-good-layout-2026-07-10

   Or from main use:
   git reset --hard local-desktop-good-layout-2026-07-10

   Only reset if you intentionally want to discard local changes.

6. Install dependencies:
   npm install

7. Create env:
   copy .env.example .env

8. Fill optional env variables:
   VITE_SUPABASE_URL=
   VITE_SUPABASE_ANON_KEY=
   VITE_KITCHEN_API_URL=
   VITE_KITCHEN_API_TOKEN=

9. Run local dev desktop:
   npm run electron:dev

10. Create desktop shortcut:
   Run:
   powershell -ExecutionPolicy Bypass -File "Create Desktop Shortcut.ps1"

11. Launch:
   Double-click "Kitchen OS - Ego Foods.lnk" on Desktop.

- Howler alarm starts if there are paid new orders
- alarm stops when no paid new orders remain

Customer pickup OTP:
- each WhatsApp order should have pickup_code/OTP
- customer should be told to share OTP with pickup person
- backend is responsible for WhatsApp message


WHATSAPP BEHAVIOR EXPECTED
==========================
WhatsApp bot backend is separate from this desktop app.
Desktop can connect to it through Kitchen API bridge.

Expected WhatsApp flow:
1. Customer sends hi/menu.
2. Bot greets and asks name first time.
3. Bot shows menu in button/list flow.
4. Customer selects items/qty.
5. Customer checks out.
6. Bot asks for payment screenshot.
7. Customer uploads screenshot.
8. Order appears in desktop dashboard as paid new.
9. Alarm rings until owner accepts/rejects.
10. Owner accepts.
11. Bot sends confirmation, pickup address, and OTP.
12. Owner marks ready.
13. Bot sends ready message.
14. Completed order can trigger thank-you/review message.

WhatsApp open/closed:
- desktop WhatsApp ON/OFF button updates backend is_open
- if closed, bot replies:
  "Orders are currently not being taken. If we resume in some time, we will update you."
- bot should not take order while closed
- reopened customers can be notified later

==========================
Swiggy import is desktop-local through Electron and Playwright.

Features:
- save Swiggy credentials locally
- use persistent browser profile
- password login where possible
- manual OTP/captcha if needed
- Import Now button
- auto import every 15 min
- default import date is current day
- incremental only
- deduplicate by full Swiggy order ID
- status/progress bar live during import
- one CSV per order date
- one item-count CSV per date

- total
- itemsDetail
